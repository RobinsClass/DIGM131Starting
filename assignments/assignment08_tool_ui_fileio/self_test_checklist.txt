Assignment 8: Self-Test Checklist
DIGM 131 - Intro to Scripting for Digital Media

INSTRUCTIONS:
    Before submitting, test each item below. Mark [X] for pass, [ ] for fail.
    If any item fails, fix it before submitting.

============================================================================
FUNCTIONALITY
============================================================================

[ ] Does the window open without errors?
[ ] Does each button perform its intended action?
[ ] Can I run the script multiple times without duplicate windows?
[ ] Does Save Settings create a JSON file I can find?
[ ] Does Load Settings restore values into the UI correctly?
[ ] Does the tool work on a fresh, empty Maya scene?

============================================================================
ERROR HANDLING
============================================================================

[ ] Does it handle empty text fields gracefully (no crash)?
[ ] Does it handle impossible slider values (if applicable)?
[ ] Does it handle missing objects (e.g., deleting something not in the scene)?
[ ] Does it handle a missing settings file when loading?
[ ] Does it handle a corrupted/invalid JSON settings file?
[ ] Does it give the user clear feedback when something goes wrong?

============================================================================
USER FEEDBACK
============================================================================

[ ] Does it confirm when objects are created (print or dialog)?
[ ] Does it confirm when objects are deleted?
[ ] Does it confirm when settings are saved?
[ ] Does it confirm when settings are loaded?
[ ] Are error messages helpful (say what went wrong and suggest a fix)?

============================================================================
CODE QUALITY
============================================================================

[ ] Is UI code separated from logic code (two files)?
[ ] Do all functions have docstrings?
[ ] Are variable names descriptive (not x, temp, thing)?
[ ] Does the code follow PEP 8 style (snake_case, proper spacing)?
[ ] Are there no hardcoded magic numbers (use named constants)?
[ ] Does it clean up after errors (no half-created objects left behind)?
